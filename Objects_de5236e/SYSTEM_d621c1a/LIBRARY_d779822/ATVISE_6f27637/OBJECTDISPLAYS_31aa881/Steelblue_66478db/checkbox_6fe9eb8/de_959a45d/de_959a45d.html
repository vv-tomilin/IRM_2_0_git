Beschreibung

Die Checkbox wechselt zwischen zwei Werten (wie die Schaltfläche "Schalter") und kann nur als Eingabe- oder Ausgabe-Steuerelement verwendet werden (siehe den Parameter "editierbar").

Sie kann in folgenden Modi verwendet werden:

  1. nur "base" festgelegt: In diesem Fall müssen der "Wert für Anzeige Zustand Ein", der "Wert der bei Ein gesetzt wird" und der "Wert der bei Aus gesetzt wird" ebenfalls definiert werden. HINWEIS: Der "Wert für Anzeige Zustand Ein" muss mit "Wert der bei Ein gesetzt wird" identisch sein.
Bei dieser Konfiguration wechselt die Checkbox bei jedem Klicken zwischen zwei Werten für die Adresse "base". Sie befindet sich im Ein-Zustand, wenn der Wert von "base" mit "Wert für Anzeige Zustand Ein" übereinstimmt; andernfalls befindet sie sich im Aus-Zustand.
  1. "base" und "Zu setzende Adresse" festgelegt:
Bei dieser Konfiguration geschieht bei jedem Klicken Folgendes:
  • Die Checkbox befindet sich im Ein-Zustand, wenn der Wert von "base" mit "Wert für Anzeige Zustand Ein" übereinstimmt; andernfalls befindet sie sich im Aus-Zustand.
  • Je nachdem, ob sich die Checkbox im Zustand "Ein" oder "Aus" befindet, wird beim Klicken die "Zu setzende Adresse" auf "Wert der bei Aus gesetzt wird" oder "Wert der bei Ein gesetzt wird" gesetzt.
  • Die Checkbox selbst ändert ihren Zustand nur (aktiviert oder nicht), wenn "base" geändert wird (Feedback-Kanal).
  1. Weder "base" noch "Zu setzende Adresse" festgelegt:
In diesem Fall wird der Trigger "valuechanged" mit dem Wert von "Wert der bei Aus gesetzt wird" oder "Wert der bei Ein gesetzt wird" aktiviert. Über diese Trigger kann ein Simple Dynamic zu dieser Checkbox hinzugefügt werden, das auf die Trigger reagiert und einen neuen Trigger aktiviert. Dieser neue Trigger kann dann von anderen Elementen verwendet werden.

Mit Hilfe einer Statusanzeige in Form eines zusätzlichen Rahmens kann zudem auch ein negativer Status des aktuellen Wertes und ein zusätzlich definierter Alarm dargestellt werden. Die Farben und Blinkintervalle werden in globalen Listen definiert.

Parameter

Dieses Steuerelement hat die folgenden allgemeinen Parameter:

  • base: Je nachdem, ob "Zu setzende Adresse" definiert ist oder nicht, folgt das Verhalten dem vorstehenden Modus 1 oder 2.
  • Alarmadresse: Optional zusätzliche Adresse eines Alarms, welcher angezeigt werden soll.
  • Beschriftung: Beschriftung der Checkbox
  • Wert für Anzeige Zustand Ein: Wert, der definiert, wann die Checkbox aktiviert ist
  • Wert der bei Ein gesetzt wird: In Abhängigkeit von der Konfiguration wird "base" oder "Zu setzende Adresse" auf diesen Wert gesetzt, wenn vom Zustand "Aus" zum Zustand "Ein" gewechselt wird.
  • Wert der bei Aus gesetzt wird: In Abhängigkeit von der Konfiguration wird "base" oder "Zu setzende Adresse" auf diesen Wert gesetzt, wenn vom Zustand "Ein" zum Zustand "Aus" gewechselt wird.
  • Statusüberwachung aktivieren: Schaltet die Anzeige des aktuellen Wertestatus ein oder aus.
  • Statustrigger feuern: Statt der Statusanzeige kann auch ein hier definierter Trigger global gefeuert werden.
Schrift:
  • Schriftart: Schriftart, die auf den Wert angewendet wird (Standard: Arial)
  • Größe: Schriftgröße, die auf den Wert angewendet wird (Standard: 12)
  • Farbe:Farbe, die auf den Wert angewendet wird (Standard: grau)
Darstellung:
  • Diese Checkbox vorwählen: Bei der Einstellung "true" wird die Checkbox beim Laden aktiviert sein, andernfalls nicht. Dies gilt nur für den Fall, dass keine "base" und keine "Zu setzende Adresse" definiert sind.
  • Füllfarbe: Füllfarbe der Checkbox (Standard: grau)
  • Füllfarbe nicht editierbar: Füllfarbe, wenn die Checkbox nicht editierbar ist (Standard: weiß)
  • Füllfarbe wenn inaktiv: Füllfarbe, wenn nicht aktiv (Standard: dunkelgrau)
  • Konturfarbe gedrückt: Farbe der Kontur, die beim Klicken in die Checkbox eingestellt wird (Standard: weiß)
  • Symbol: Symbol, das bei aktivierter Checkbox angezeigt wird; weitere Symbole können ausgewählt werden aus "Object Displays (ATVISE)/Symbols/20x20 (Standard: x)
  • Symbolfarbe: Farbe des zu verwendenden Symbols (Standard: schwarz)
  • Fokus Konturfarbe: Farbe der Kontur, die beim Fokus der Schaltfläche eingestellt wird (Standard: weiß)
  • Statushinweis: Spezifiziert die Darstellung des Status über eine definierte globale Liste (Standard: Globaler Parameter atvStatusIndication)
  • Alarmhinweis: Spezifiziert die Darstellung des Alarms überber eine definierte globale Liste (Standard: Globaler Parameter atvAlarmIndication)
Optionen:
  • Zu setzende Adresse: Das Verhalten folgt dem vorstehenden Modus 2.
  • editierbar: Bei der Einstellung "Yes" kann die Checkbox auch als Eingabefeld verwendet werden, d. h. die Checkbox kann aktiviert werden. Bei der Einstellung "No" kann sie nur als Ausgabefeld verwendet werden.
  • Konsistenzgruppe: Name der Konsistenzgruppe
  • Tab-Index: Bestimmt die Reihenfolge, in der mit der Tabulatortaste zwischen den einzelnen Object Displays gewechselt werden kann
  • Tooltip: Zeigt eine Tooltip Box mit dem eingegebenen Text an (Standard HTML-Elemente können auch eingegeben werden)
Sicherheit:
  • Benötigtes Recht: Der Benutzer muss über dieses Recht verfügen, um diese Checkbox aktivieren zu können.
  • Aktivierungsadresse: Stimmt der Wert dieser Adresse mit dem "Aktivierungswert" überein, ist die Checkbox aktiv, so dass sie aktiviert werden kann.
  • Aktivierungswert: Wert, von dem die Checkbox aktiviert wird
HINWEIS: Die Aktivierung der Checkbox kann nur über das benötigte Recht konfiguriert werden oder nur über die Aktivierungsadresse und den Aktivierungswert oder über beide. Werden beide Einstellungen vorgenommen, muss der Benutzer das benötigte Recht besitzen und der Wert der "Aktivierungsadresse" muss mit dem "Aktivierungswert" identisch sein, andernfalls wird die Checkbox nicht aktiviert.

Beispiel

Für den vorstehend beschriebenen Modus 2 gehen wir von den folgenden Parametern aus:
  • base: node_feedback
  • Wert für Anzeige Zustand Ein: 1
  • Wert der bei Ein gesetzt wird: 1
  • Wert der bei Aus gesetzt wird: 0
  • Zu setzende Adresse: node_set
Weiterhin gehen wir davon aus, dass die Werte von "base" und "Zu setzende Adresse" auf 0 eingestellt sind.
  1. Bei diesen Einstellungen wird die Checkbox nicht aktiviert.
  2. Beim Klicken wird "node_set" auf 1 (= "Wert der bei Ein gesetzt wird") gesetzt; die Checkbox ist noch immer nicht aktiviert (der Wert von "base" weicht noch immer vom "Wert für Anzeige Zustand Ein" ab).
  3. Sobald "base" auf 1 gesetzt ist (Feedback-Kanal von SPS), ist die Checkbox aktiviert" (der Wert von "base" ist jetzt mit dem "Wert für Anzeige Zustand Ein" identisch).
  4. Beim Klicken wird "node_set" auf 0 (= "Wert der bei Aus gesetzt wird") gesetzt; die Checkbox ist noch immer aktiviert (der Wert von "base" ist noch immer mit dem "Wert für Anzeige Zustand Ein" identisch).
  5. Sobald "base" auf 0 gesetzt ist (Feedback-Kanal von SPS), ist die Checkbox nicht mehr aktiviert" (der Wert von "base" weicht jetzt vom "Wert für Anzeige Zustand Ein" ab).
