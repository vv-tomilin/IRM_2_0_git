Beschreibung

Dieses Display verbindet die Displays Highcharts, Toolbar und Legende. Es kann als Beispiel dienen oder unverändert in einem Projekt verwendet werden.

Die unten folgenden Parameter werden an die einzelnen Displays weitergereicht, sodass jedes Display dieselbe Chartinstanz als Basis besitzt. Dadurch wird sichergestellt, dass Änderungen im Konfigurationsdialog oder der Legende das richtige Diagramm ansprechen.

Parameter

Die nachstehenden allgemeinen Parameter sind verfügbar:

  • Diagrammname: Eindeutiger Name zur Identifizierung der Chartinstanz
  • Diagrammgruppe: Dient dem Hinzufügen unterschiedlicher Chartinstanzen zu einer Gruppe.
  • Standardmodule überspringen: Bestimmt, ob die Standardmodule für Highcharts geladen werden sollen.
  • Zusatzmodule laden: Definiert welche Module zusätzlich geladen werden sollen. (Beispiel: "highcharts/modules/exporting.js,highcharts/datasource/opcua.js,highcharts/atviseModules/datahandler.js,highcharts/atviseModules/measuringcursor.js,highcharts/atviseModules/utils.js,highcharts/atviseModules/downsample.js")
  • Konfigurationsadresse: Node (Adresse), welche eine vordefinierte Konfiguration enthält.
  • Datei bei Speicherung im Dateisystem: Datei, welche eine vordefinierte Konfiguration neu hält, wenn Speichern im Dateisystem eingestellt ist.
  • Konfigurationsname: Name der vordefinierten Konfiguration.
  • Speichermethode: Legt fest, ob Konfigurationen in Node(s) oder dem Dateisystem gespeichert werden.
  • Komprimierung: Legt fest, ob die Konfigurationsdaten beim Speichern komprimiert werden.
  • Datenquelle: Definiert den Typ der Datenquelle, welche über ein entsprechendes Javascript-Modul geladen wird.
  • Zurück/Vor Schrittweite in Prozent: Legt die Schrittweite für das Zurück/Vor-Blättern in historischen Daten fest.

Außerdem können über entsprechende Parameter die benötigten Rechte zur Anzeige des Konfigurationsdialogs ("Optionen anzeigen"), zum Laden und Speichern der Konfiguration und zur Anzeige des Exportmenüs vergeben werden:

  • Benötigtes Recht: Der Benutzer muss über dieses Recht verfügen, um die Funktion ausführen zu können.
  • Aktivierungsadresse: Stimmt der Wert dieser Adresse mit dem "Aktivierungswert" überein, ist die Funktion aktiviert.
  • Aktivierungswert: Wert, von dem die Funktion aktiviert wird
