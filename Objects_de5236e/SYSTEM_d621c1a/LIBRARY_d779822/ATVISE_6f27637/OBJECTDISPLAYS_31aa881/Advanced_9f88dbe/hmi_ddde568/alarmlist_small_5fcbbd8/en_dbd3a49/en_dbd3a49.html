Description

This is a version for smaller displays of tablets or smarthphones.

The alarmlist is an advanced control that displays the current active alarms of the system and defined alarm conditions in form of a table (including filter possibilities).

Views

There are several views which can be preset through display parameters:

  • alarms: All triggered alarms are listed here.
  • all: Alle triggered alarms and all defined alarm conditions are listed here.
  • shelved: All shelved alarms and alarm conditions are listed here.
  • suppressed: All suppressed alarms and alarm conditions are listed here.
  • disabled: All disabled alarms and alarm conditions are listed here.

The views can be selected in the head of the atvise standard visualisation or through the "Open alarm list" button.

Priority

The (column) priority defined in the global list is used to display different columns on different devices (screen sizes). Columns of priority 1 are dispalyed on every device, columns of priority 2 and higher are hidden on small dispalys (smartphones), columns of priority 3 and higher are only visibile on bigger displays (desktop computers).

Filter

On top of the alarm list display you can find a set of filter elements. After changing the input elements and clicking the "Apply filter" button, the new filter is applied to the alarmlist.

  • Min. Priority: "Greater equal" filter by severity (defined at the alarm category).
  • Display: Filter by the attribute "display" that can be defined in the alarm configuration.
  • Group: Filter for alarms of one or more "alarm groups". Multiple alarm groups can be entered comma- or semicolon-seperated.
  • Alarm source: Filter by the data source of alarms.
  • Alarm condition: Filter by the node of the alarm condition.
  • Ackn. state: Filter by ALL or one of the alarm states:
    • ACTIVE!
    • ACTIVE! unconf.
    • active ack.
    • active conf.
    • INACTIVE!
    • INACTIVE! conf.
    • (IN)ACTIVE!
    • (IN)ACTIVE! conf.
  • User: Filter by username for already acknowledged alarms.
  • Event text: Filter by the alarm text that has to be defined in the alarm configuration.
  • Comment: Filter by the comment of the alarm.
  • Alarm or condition: Filter by triggered alarms, alarm conditions or both.
  • State: Filter by states of alarms and conditions. The options include triggered alarms, "Suppressed", "OneShotShelved" and "TimeShelved" status. This filter field is only available in the "All" view.

By default two rows of filter fields are shown. By clicking the "Advanced filters" button the residual filter fields are displayed.

By clicking the button "Reset filter" you can unset all applied filters and display the default settings.

Alarms

Each alarm can be acknowledged.

Confirmable alarms can be confirmed after acknowledging.

A link to the corresponding display will be avaiable if the display was specified in the configuration.

For retrigger-alarms each triggering state change causes an distinct alarm in the table.

The alarm category "Acknowledge active and inactive" with explicit acknowledgement is displayed as a pair of alarms with identical background color. The inactive alarm state is shown on top of the active alarm state. Before the inactive state can be acknowledged the acknowledgement of the active state is required.

For historized alarms please use the "history list" display.

Actions

  • acknowledge: An alarm is acknowledged.
  • confirm: An acknowledged alarm is confirmed.
  • acknowledge with comment: An alarm is acknowledged via a pop up dialog which allows to pass a comment.
  • confirm with comment: An acknowledged alarm is confirmed via a pop up dialog which allows to pass a comment.
  • shelve: Alarm or alarm condition is being shelved. The user cannot see alarms and conditions in the regular alarm list any longer.
    • one shot shelve: This action can only be applied to alarms and not to alarm conditions. The alarm is shelved until it is unshelved or the maximum shelving time set in the MaxTimeShelved variable is reached.
    • timed shelve: This action can be applied to alarms and alarm conditions. The alarm or alarm condition is shelved for a time period set by the user in a pop up dialog.
  • unsuppress: Alarm or alarm condition is being unsuppressed. Suppressing happens internally on the server. Suppressed alarms are not shown in the regular alarm list.
  • disable: Alarm condition and associated active alarms are being disabled. No further alarms and events are generated.

Detail table

Further details of the alarms and conditions can be displayed in a foldable detail table.

The fields shown in the detail table can be configured through a global list which's address can be selected in the display parameters.

If new alarms appear in the alarm list, the rows of the shown alarms can jump to different positions unfavorably and display the detail table on a different position. To fix the detail table or certain rows, the favored rows can be marked using the checkboxes on the left. After that you can use the filter to view only the marked rows.

Parameters

  • base für Displays: base für verlinkte Displays (default: base)
  • view: Default view of alarmlist (default: alarms). The available options are:
    • all: Displays all alarms and alarm conditions
    • alarms: Displays triggered alarms
    • shelved: Displays shelved alarms and alarm conditions
    • suppressed: Displays suppressed alarms and alarm conditions
    • disabled: Displays disabled alarm conditions
Table:
  • table ID: Unique table identifier (default: "alarm_table")
  • table title: Title of the table (default: "T{Alarm list}")
  • table header height: Height of the table header (default: 35)
  • table row height: Height of a table row (default: 35)
Columns:

The table's columns can be configured through a global list which's address can be selected here. The possiblee parameters in the global list are:

  • alignmentMain: Text alignment
  • columnName: Column name
  • field: Field name which is delivered to the table
  • filter: Activate filter for the column
  • fontSizeDetail: Font size of detail view
  • fontSizeMain: Font size of table
  • orderDetail: Order of fields in the detail view
  • orderMain: Order of table's columns
  • priority: Priority in size dependent viewing
  • sortable: Columns is sortable
  • textoption: Appearance of the text. The available options are:
    • normal
    • italic
    • small-caps
    • bold
    • bold-italic
    • bold-small-caps
    • bolder
    • bolder-small-caps
    • lighter
    • lighter-italic
    • lighter-small-caps
  • type: Type of column values. The available options are:
    • datetime: Date/time field. Add ", ms" to show milliseconds.
    • number: Number field. Add ", [number]" to show [number] decimals.
    • string: Text field
  • visibleDetail: Visibility in detail view
  • visibleMain: Visibility of column in main table
  • widthMain: Width of column in main table
Font:
  • font family: font family applied to the table (default: Arial)
  • font color: color which will be applied to labels and values (default: global parameter atvFontColor)
  • table header font size: font size applied to the table header (default: 12)
  • table row font size: font size applied to the table rows (default: 12)
  • table detail font size: font size applied to the detail area (default: 12)
  • table footer font size: font size applied to the table footer (default: 12)
Appearance:
  • border color: color of borders (default: global parameter atvBorderColor)
  • fill color: fill color of the top background area (default: global parameter atvFillColor)
  • symbol color: color of symbols (default: global parameter atvSymbolColor)
  • global color config: global color configuration (default: global list atvObjectDisplays.Tables.ColorConfig)
  • alarm indication appearance: appearance of alarm indication (default: global parameter atvAlarmIndication)
  • custom theme: Name of the cusom theme (default: atviseTheme)
  • shelve symbol: symbol for shelving
  • shelve once symbol: symbol for shelve once
  • timed shelve symbol: symbol for timed shelve
  • suppress symbol: symbol for suppressing
  • enable symbol: symbol for enabling
  • acknowledge symbol: symbol for acknowledging
  • confirm symbol: symbol for confirming
  • acknowledge with comment symbol: symbol for acknowledging with comment
  • confirm with comment symbol: symbol for confirming with comment
  • acknowledge inactive symbol: inactive symbol for acknowledging
  • confirm inactive symbol: inactive symbol for confirming
  • acknowledge with comment inactive symbol: inactive symbol for acknowledging with comment
  • confirm with comment inactive symbol: inactive symbol for confirming with comment
  • action button symbol: symbol for the action button
Options:
  • query filter: Shows or hides the query filter (default: true)
  • paging: Activate paging (default: false)
  • filter bar enabled: Show filter bar (default: true)
  • filter regular expressions: Regular expressions can be used for the filter (default: false)
  • filter use placeholder: Activation of the "*" placeholder in the search function (default: false)
  • filter case sensitive: Filter works case sensitive (default: false)
  • filter min. input length: Minimal input length before the filter starts (default: 1)
  • stick parent and child together: Children are always listed together with their parent (default: true)
  • cell linebreak: Too long text will be broken to new lines. If there is no more space the text will be cut ending with "..." (default: false)
  • columns reorderable: Columns can be reorderedy (default: true)
  • multiselect on cell click: Lines can be selected by clicking them (default: true)
Menu options:
  • show menu: Display the menu (default: true)
  • custom menu title: Title of the custom menu (default: "T{Export CSV}")
  • picker menu title: Title of the picker menu (default: "Show/Hide")
  • show picker menu: Display the picker menu (default: true)
  • menu width: Width of the menu (default: 250)
  • menu height: Height of the menu (default: 300)

Security.
  • right for acknowledge: if the right for acknowledge is being set, a user can only accept an alarm if he has the right for acknowledge. In the case that a user does not have the right for acknowledge, the according symbols and links will be inactive. If no right for acknowledge is specified, alarms can be accepted by any user.
  • right for confirm: if the right for confirm is being set, a user can only confirm an alarm if he has the right for confirm. In the case that a user does not have the right for confirm, the according symbols and links will be inactive. If no right for confirm is specified, alarms can be confirmed by any user.
  • right for (un)shelve: if the right for (un)shelve is being set, a user can only shelve or unshelve alarms and conditions if he has the right for (un)shelve. In the case that a user does not have the right for (un)shelve, the according symbols and links will be inactive. If no right for (un)shelve is specified, alarms or conditions can be shelved or unshelved by any user.
  • right for unsuppress: if the right for unsuppress is being set, a user can only unsuppress an alarm condition if he has the right for unsuppress. In the case that a user does not have the right for unsuppress, the according symbols and links will be inactive. If no right for unsuppress is specified, alarm conditions can be unsuppressed by any user.
  • right for disable/enable: if the right for disable/enable is being set, a user can only disable or enable alarm conditions if he has the right for disable/enable. In the case that a user does not have the right for disable/enable, the according symbols and links will be inactive. If no right for disable/enable is specified, alarm conditions can be enabled or disabled by any user.
  • right for CSV export: if the right for CSV export is being set, a user can only use the CSV export an alarm if he has the right for CSV export. In the case that a user does not have the right for CSV export, the according links will be inactive. If no right for CSV export is specified, CSV export can be carried out by any user.
