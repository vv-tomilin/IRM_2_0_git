Beschreibung

Bei der Alarmliste handelt es sich um ein erweitertes Steuerelement, das die derzeit aktiven Systemalarme und/oder definierte Alarmbedingungen in Tabellenform anzeigt und Filtermöglichkeiten bietet.

Ansichten

Es gibt verschiedenen Ansichten, die über Displayparameter voreingestellt werden können:

  • Alarme: Hier werden alle ausgelösten Alarme dargestellt.
  • Alle: Alle ausgelösten Alarme und alle definierten Alarmbedingungen werden dargestellt.
  • Zurückgestellt: Alle zurückgestellten Alarme und Alarmbedingungen werden dargestellt.
  • Unterdrückt: Alle unterdrückten Alarme und Alarmbedingungen werden dargestellt.
  • Ausgeschaltet: Alle augeschalteten Alarmbedingungen werden dargestellt.

Die Ansichten können in der Kopfzeile der atvise Standardvisu auch über die "Alarmliste Öffnen" Schaltfläche ausgewählt werden.

Priorität

Die in der globalen Liste definierte (Spalten-)Priorität wird zur Anzeige von unterschiedlichen Spalten auf verschiedenen Endgeräten genutzt. Spalten mit der Priorität 1 werden auf jedem Gerät angezeigt, Spalten ab der Priorität 2 werden auf kleinen Displays (Smartphones), Spalten ab der Priorität 3 auf mittleren Displays (Tablets) ausgeblendet.

Filter

Über Filterelemente im Kopf der Alarmliste kann eine verfeinerte Auswahl an Alarmen oder Alarmbedingungen getroffen werden. Nach der Änderung der Eingabeelemente löst das Betätigen der "Filtern"-Schaltfläche eine neue Filterung aus.

  • Min. Prio.: "Größer gleich"-Filterung nach Priorität (definiert in der Alarmkategorie)
  • Display: Filterung nach dem Attribut "Display", das in der Alarmkonfiguration definiert werden kann
  • Gruppe: Filterung nach den Alarmen einer oder mehrerer "Alarmgruppen". Mehrere Alarmgruppen können durch Beistrich oder Strichpunkt getrennt eingegeben werden.
  • Alarmquelle: Filterung nach der Datenquelle des Alarms
  • Alarmbedingung: Filterung nach einer bestimmten Alarmbedingungsadresse
  • Quitt.-Status: Filterung nach ALLEN oder einem der Alarmzustände:
    • AKTIV!
    • AKTIV! unbest.
    • aktiv quitt.
    • aktiv unbest.
    • INAKTIV!
    • INAKTIV! unbest.
    • (IN)AKTIV!
    • (IN)AKTIV! unbest.
  • Benutzer: Filterung nach Benutzername für bereits quittierte Alarme
  • Ereignistext: Filterung nach dem Alarmtext, der in der Alarmkonfiguration definiert sein muss
  • Kommentar: Filterung nach Kommentaren von Alarmen
  • Alarm od. Alarmbedingung: Filterung nach ausgelösten Alarmen, Alarmbedingungen oder beidem
  • Status: Filterung nach Zuständen von Alarmen und Bedingungen. Zur Auswahl stehen ausgelöste Alarme, "Unterdrückt"-, "OneShotShelved"- und "TimeShelved"-Status. Dieses Filterfeld steht nur in der "Alle"-Ansicht zur Verfügung.

Standardmäß werden zwei Zeilen mit Filterfeldern angezeigt. Über die Schaltfläche "Erweiterte Filter" können die restlichen Filterfelder eingeblendet werden.

Mit der Schaltfläche "Filter zurücksetzen" können Sie alle angewandten Filter deaktivieren und die Alarmliste auf die Standardparameter setzen.

Alarme

Jeder Alarm kann quittiert werden.

Bestätigbare Alarme können nach dem Quittieren bestätigt werden.

Sofern ein Display in der Konfiguration definiert wurde, wird ein Link für die Navigation zu dem entsprechenden Display angezeigt.

Bei Retrigger-Alarmen wird für jede neue Auslösung ein eigener Alarm dargestellt.

Die Alarmkategorie "Kam und Ging quittierpflichtig" wird als ein aufeinanderfolgendes Alarmpaar in der Tabelle dargestellt. Der Ging-Alarm wird chronologisch immer vor dem Kam-Alarm angezeigt und es muss immer der KAM-Teil zuerst quittiert werden.

Auf historische Alarme können Sie über das Display "Historische Liste" zugreifen.

Aktionen

  • Quittieren: Ein Alarm wird quittiert.
  • Bestätigen: Ein quittierter Alarm wird bestätigt.
  • Quittieren mit Kommentar: Ein Alarm wird über einen Popup-Dialog quittiert, in dem auch ein Kommentar mitgegeben werden kann.
  • Bestätigen mit Kommentar: Ein quittierter Alarm wird über einen Popup-Dialog bestätigt, in dem auch ein Kommentar mitgegeben werden kann.
  • Zurückstellen: Alarm oder Alarmbedingung wird zurückgestellt. Der Benutzer kann Alarme dieser Bedingung nicht mehr in der normalen Alarmliste sehen.
    • Einmalig zurückstellen: Diese Aktion kann nur auf Alarme und nicht auf Alarmbedingungen angewandt werden. Der Alarm wird zurückstellt bis das Zurückstellen durch den Benutzer beendet wird oder die in der MaxTimeShelved-Variable festgelegte maximale Zurückstelldauer erreicht wurde.
    • Befristet zurückstellen: Diese Aktion kann auf Alarme und Alarmbedingungen angewandt werden. Der Alarm bzw. die Alarmbedingung wird für eine vom Benutzer in einem PopUp-Dialog festgelegte Zeit zurückgestellt.
  • Unterdrücken rückgängig machen: Alarm oder Alarmbedingung wird unterdrückt. Zurückstellen erfolgt nur intern am Server. Unterdrückte Alarme werden in der normalen Alarmliste nicht angezeigt.
  • Ausschalten: Alarmbedingung und dazugehörige aktive Alarme werden ausgeschalten. Es werden keine Alarme und Events mehr erzeugt.

Detailbereich

Weitere Details der Alarme und Alarmbedingungen können über einen ausklappbaren Detailbereich angezeigt werden.

Über eine globale Liste, die über die Displayparameter ausgewählt werden kann, können die im Detailbereich angezeigten Daten konfiguriert werden.

Erscheinen neue Alarme in der Alarmliste, so kann ein unvorteilhaftes Springen der angezeigten Alarme auftreten, das den Detailbereich an einer anderen Stelle erscheinen lässt. Möchte man den Detailbereich oder bestimmte Zeilen fixieren, so geschieht dies am besten dadurch, dass man die gewünschten Zeilen markiert und eine Filterung auf markierte Zeilen durchführt.

Parameter

  • base für Displays: base für verlinkte Displays (Standard: base)
  • Startansicht: Standard-Ansicht beim Laden der Alarmliste (Standard: alarms) Die verfügbaren Optionen sind:
    • all: Ansicht zur Anzeige aller Alarme und Alarmbedingungen
    • alarms: Ansicht zur Anzeige von aktiven Alarmen
    • shelved: Ansicht zur Anzeige zurückgestellter Alarme und Alarmbedingungen
    • suppressed: Ansicht zur Anzeige unterdrückter Alarme und Alarmbedingungen
    • disabled: Ansicht zur Anzeige ausgeschalteter Alarmbedingungen
Tabelle:
  • Tabellen-ID: Eindeutige Bezeichnung der Tabelle (Standard: "alarm_table")
  • Tabellentitel: Titel der Tabelle (Standard: "T{Alarm list}")
  • Tabellenkopf Höhe: Höhe des Tabellenkopfs (Standard: 30)
  • Tabellenzeile Höhe: Höhe einer Tabellenzeile (Standard: 20)
Spalten:

Über eine globale Liste, die hier ausgewählt werden kann, können die Spalten der Tabelle konfiguriert werden. Die einstellbaren Parameter der globalen Liste sind:

  • alignmentMain: Ausrichtung des Textes
  • columnName: Name der Spalte
  • field: Feldname, der an die Tabelle geliefert wird
  • filter: Filter für die Spalte aktivieren
  • fontSizeDetail: Schriftgröße der Detailanzeige
  • fontSizeMain: Schriftgröße der Tabelle
  • orderDetail: Anzeigereihenfolge in der Detailanzeige
  • orderMain: Anzeigereihenfolge der Tabellenspalten
  • priority: Priorität in größenabhängiger Anzeige
  • sortable: Spalte ist sortierbar
  • textoption: Darstellung des Textes. Die verfügbaren Optionen sind:
    • normal
    • italic
    • small-caps
    • bold
    • bold-italic
    • bold-small-caps
    • bolder
    • bolder-small-caps
    • lighter
    • lighter-italic
    • lighter-small-caps
  • type: Typ der Werte in der Spalte. Die verfügbaren Optionen sind:
    • datetime: Datum/Zeit-Feld. Der Zusatz ", ms" stellt die Zeit in der Tabelle inklusive Millisekunden dar.
    • number: Zahlen-Feld. Der Zusatz ", [Anzahl]" stellt Zahlen mit [Anzahl] Kommastellen dar.
    • string: Text-Feld
  • visibleDetail: Sichtbarkeit im Detailbereich
  • visibleMain: Sichtbarkeit als Spalte in der Haupttabelle
  • widthMain: Breite der Spalte in der Haupttabelle
Schrift:
  • Schriftart: Die in der Tabelle verwendete Schriftart (Standard: Arial)
  • Schriftfarbe: Farbe, die auf Beschriftungen angewendet wird (Standard: Globaler Parameter atvFontColor)
  • Tabellenkopf Schriftgröße: Die im Tabellenkopf angewandte Schriftart (Standard: 12)
  • Tabellenzeile Schriftgröße: Die in den Tabellenzeilen angewandte Schriftart (Standard: 12)
  • Detailbereich Schriftgröße: Die in der Detailansicht angewandte Schriftart (Standard: 12)
  • Tabellenfuß Schriftgröße: Die im Tabellenfuß angewandte Schriftart (Standard: 12)
Darstellung:
  • Rahmenfarbe: Farbe von Rahmen (Standard: Globaler Parameter atvBorderColor)
  • Füllfarbe: Füllfarbe der oberen Hintergrundfläche (Standard: Globaler Parameter atvFillColor)
  • Symbolfarbe: Farbe von Symbolen (Standard: Globaler Parameter atvSymbolColor)
  • Globale Farbkonfiguration: In einer globalen Liste gespeicherte Farbkonfiguration (Standard: Globale Liste atvObjectDisplays.Tables.ColorConfig)
  • Alarmhinweis: Spezifiziert die Darstellung des Alarms überber eine definierte globale Liste (Standard: Globaler Parameter atvAlarmIndication)
  • Benutzerdefiniertes Theme: Name des benutzerdefinierten Themes (Standard: atviseTheme)
  • Symbol Zurückstellen: Symbol für das Zurückstellen
  • Symbol Einmal Zurückstellen: Symbol für das Einmal Zurückstellen
  • Symbol Befristet Zurückstellen: Symbol für das Befristet Zurückstellen
  • Symbol Unterdrücken: Symbol für das Unterdrücken
  • Symbol für Einschalten: Symbol für das Einschalten
  • Symbol Quittieren: Symbol für das Quittieren
  • Symbol Bestätigen: Symbol für das Bestätigen
  • Symbol Quittieren mit Kommentar: Symbol für das Quittieren mit Kommentar
  • Symbol Bestätigen mit Kommentar: Symbol für das Bestätigen mit Kommentar
  • Symbol Quittieren inaktiv: Inaktives Symbol für das Quittieren
  • Symbol Bestätigen inaktiv: Inaktives Symbol für das Bestätigen
  • Symbol Quittieren mit Kommentar inaktiv: Inaktives Symbol für das Quittieren mit Kommentar
  • Symbol Bestätigen mit Kommentar inaktiv: Inaktives Symbol für das Bestätigen mit Kommentar
  • Symbol Aktionenschaltfläche: Symbol für Aktionenschaltfläche
Optionen:
  • Abfragefilter: Blendet die Abfragefilter ein oder aus (Standard: true)
  • Paging: Aktivieren des Pagings (Standard: false)
  • Filterleiste sichtbar: Einblenden der Filterleiste (Standard: true)
  • Filter reguläre Ausdrücke: Reguläre Ausdrücke für das Filtern verwenden (Standard: false)
  • Filter mit Platzhalter: Aktivierung des Platzhalters "*" in der Suchfunktion (Standard: false)
  • Filter Groß-/Kleinschreibung: Groß- und Kleinschreibung beim Filtern beachten (Standard: false)
  • Filter min. Eingabelänge: Eingabelänge, ab der ein Filtervorgang ausgelöst wird (Standard: 1)
  • Parent und Child zusammen anzeigen: Unteralarme werden immer zusammen mit ihren Elternalarmen gelistet (Standard: true)
  • Zeilenumbruch in Zelle: Zu langer Text wird innerhalb der Zeile umgebrochen, bei zu wenig Platz mit "..." als Endung abgeschnitten (Standard: false)
  • Spalten verschiebbar: Das Ändern der Spaltenreihenfolge wird ermöglicht (Standard: true)
  • Mehrfachselektion bei Klick: Das Selektieren von Zeilen ist durch Klicks auf die Zeilen möglich (Standard: true)
Menüoptionen:
  • Menü anzeigen: Das Menü wird angezeigt (Standard: true)
  • Titel benutzerdefiniertes Menü: Der Titel des benutzerdefinierten Menüs (Standard: "T{Export CSV}")
  • Pickermmenü Titel: Der Titel des Pickermenüs (Standard: "Show/Hide")
  • Pickermmenü anzeigen: Das Pickermenü wird angezeigt (Standard: true)
  • Menübreite: Die Breite des Menüs
  • Menühöhe: Die Höhe des Menüs

Sicherheit:
  • Benötigtes Recht für Quittieren: Wird hier ein Recht ausgewählt, so kann ein Benutzer einen Alarm nur dann quittieren, wenn er das angegebene Recht besitzt. Ansonsten sind Symbole und Links zum Quittieren inaktiv. Wenn kein Recht ausgewählt wird, so kann jeder Benutzer Alarme quittieren.
  • Benötigtes Recht für Bestätigen: Wird hier ein Recht ausgewählt, so kann ein Benutzer einen Alarm nur dann bestätigen, wenn er das angegebene Recht besitzt. Ansonsten sind Symbole und Links zum Bestätigen inaktiv. Wenn kein Recht ausgewählt wird, so kann jeder Benutzer Alarme bestätigen.
  • Benötigtes Recht für Zurückstellen: Wird hier ein Recht ausgewählt, so kann ein Benutzer Alarme und Alarmbedingungen nur dann zurückstellen, wenn er das angegebene Recht besitzt. Ansonsten sind Symbole und Links zum Zurückstellen inaktiv. Wenn kein Recht ausgewählt wird, so kann jeder Benutzer Alarme und Alarmbedingungen zurückstellen.
  • Benötigtes Recht für Unterdrücken rückg.: Wird hier ein Recht ausgewählt, so kann ein Benutzer unterdrückte Alarmbedingungen nur dann wieder freigeben, wenn er das angegebene Recht besitzt. Ansonsten sind Symbole und Links zum Freigeben unterdrückter Alarmbedingungen inaktiv. Wenn kein Recht ausgewählt wird, so kann jeder Benutzer unterdrückte Alarmbedingungen freigeben.
  • Benötigtes Recht für Aus- und Einschalten: Wird hier ein Recht ausgewählt, so kann ein Benutzer Alarmbedingungen nur dann aus- oder einschalten, wenn er das angegebene Recht besitzt. Ansonsten sind Symbole und Links zum Aus- und Einschalten inaktiv. Wenn kein Recht ausgewählt wird, so kann jeder Benutzer Alarmbedingungen aus- oder einschalten.
  • Benötigtes Recht für CSV-Export: Wird hier ein Recht ausgewählt, so kann ein Benutzer CSV-Exporte durchführen, wenn er das angegebene Recht besitzt. Ansonsten sind Links zum Durchführen von CSV-Exporten inaktiv. Wenn kein Recht ausgewählt wird, so kann jeder Benutzer CSV-Exporte durchführen.
