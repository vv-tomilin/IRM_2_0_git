Beschreibung

Diese Version der historischen Liste wurde speziell für kleine Displays von Tablets und Smartphones entworfen.

Bei der historischen Liste handelt es sich um ein erweitertes Steuerelement, welches auf der Basis einer an den Server gerichteten Abfrage historische Daten angezeigt. Vier Arten von Daten werden angezeigt: Alarme, Wertänderungen, Aggregate und (OPC-UA) Events.

Bitte beachten: Für die Archivierung von Alarmen und Events muss eine Archivgruppe "Event" und für die Archivierung von Aggregaten eine Archivgruppe "Aggregate" zu den historischen Archiven hinzugefügt werden.

Filter

Im oberen Bereich der historischen Liste befindet sich eine Gruppe von Filterelementen. Mit der Schaltfläche "Erweiterte Filter" können zusätzliche Filter angezeigt werden. Hier kann eine Kombination von Filtern festlegt und durch klicken auf die Schaltfläche "Filter anwenden" auf die Liste angewendet werden. Mit der Schaltfläche "Zurücksetzen" können Sie wieder alle Filter auf die standardmäßigen Parameter setzen.

  • Von: Startzeit für die Abfrage.
    Standard: 1 Minute in die Vergangenheit bzw. die Startzeit in Sekunden des vordefinierten Parameters.
  • Bis: Das Enddatum für die Abfrage (Standard: jetzt).
  • Typ: Filterung nach Typ: Alarme, Aggregate, Events und Werte

Anmerkung: Alternativ zur Auswahl des Zeitraumes über "Von-Bis", kann auch auf die Abfrage von einem Start- oder Endzeitpunkt umgeschaltet werden.

Für Alarme, Events und Werte gelten zusätlich die folgenden Parameter:

  • Adresse: Filterung nach dem Datenpunkt (Adresse) des Eintrags in der historischen Liste.
  • Wert: Filterung nach dem Wert in der historischen Liste.
  • Archiv: Filterung nach Archiv.
  • Kommentar: Filterung nach Kommentar.
  • Benutzername: Nach definiertem Benutzernamen filtern.
  • Ereignistext: Filterung nach der Alarmmeldung und Ereignistexten in der historischen Liste.
  • Min. Priorität: "Größer gleich"-Filterung nach Schweregrad (definiert in der Alarmkategorie bzw. den Events).
  • Gruppe: Filterung nach zugewiesener Gruppe.
  • Liveansicht: Aktiviert die Liveanzeige.

Ist die Checkbox "Live" aktiviert, werden aktuelle Wertänderungen in der Liste angezeigt, sobald sie auftreten. Hier kann nur nach Typ gefiltert werden.

Für Aggregate gelten die folgenden Parameter

  • Adresse: Filterung nach dem Datenpunkt (Adresse) des Eintrags in der historischen Liste.
  • Wert: Filterung nach dem Wert in der historischen Liste.
  • Archiv: Filterung nach Archiv.
  • Aggregat: Filterung nach definiertem Aggregat.

Bitte beachten: Die Schaltfläche "Filter anwenden" verwendet für die Server-Abfrage die vorgenommenen Filtereinstellungen. Bei einem langen Filterzeitraum (z. B. Daten aus einer Woche) werden möglicherweise sehr große Datenmengen abgerufen, was einige Zeit in Anspruch nehmen kann.

Wird über die Combobox eine Aggregatvorlage abgefragt (nur bei kompatiblen Servern, z.B. atvise Scada), werden für die Serveranfrage die Vorlagenparameter verwendet (Typ, Einheit und Intervall). Das kann dazu führen, dass ebenfalls Ergebnisse von anderen Aggregatvrolagen mit den gleichen Parametern zurückgeliefert und anzeigt werden.

Priorität

Die in der globalen Liste definierte (Spalten-)Priorität wird zur Anzeige von unterschiedlichen Spalten auf verschiedenen Endgeräten genutzt. Spalten mit der Priorität 1 werden auf jedem Gerät angezeigt, Spalten ab der Priorität 2 werden auf kleinen Displays (Smartphones), Spalten ab der Priorität 3 auf mittleren Displays (Tablets) ausgeblendet.

Parameter

  • Tabellen-ID: Eindeutige Bezeichnung der Tabelle (Standard: "history_table")
  • Tabellenkopf Höhe: Höhe des Tabellenkopfes in px (Standard: 35)
  • Tabellenzeile Höhe: Höhe der Tabellenzeile in px (Standard: 30)
Spalten:
  • Konfiguration: Die Spalten werden mit Hilfe einer globalen Liste konfiguriert. In dieser globalen Liste werden Spaltenname, Sichtbarkeit, Breite, Darstellung, etc. definiert. Ein Standardvorlage ist in atvise inkludiert. (Standard: atvObjectDisplays.Historylist.Columns)
Schrift:
  • Schriftart: Schriftart, welche für in der gesamten Tabelle verwendet wird (Standard: Arial)
  • Schriftfarbe: Schriftfarbe, welche in der gesamten Tabelle angewendet wird (Standard: Globaler Parameter atvFontColor)
  • Tabellenkopf Schriftgröße: Schriftgröße für den Tabellenkopf (Standard: 12)
  • Tabellenzeile Schriftgröße: Schriftgröße für die Tabellenzeile (Standard: 12)
  • Detailbereich Schriftgröße: Schriftgröße für den aufklappbaren Detailbereich (Standard: 12)
  • Tabellenfuß Schriftgröße: Schriftgröße für den Tabellenfuß (Standard: 12)
Darstellung:
  • Füllfarbe: Füllfarbe der oberen Hintergrundfläche (Standard: Globaler Parameter atvFillColor).
  • Rahmenfarbe: Farbe derTabellenrahmen (Standard: Globaler Parameter atvBorderColor).
  • Symbolfarbe: Farbe von Symbolen (Standard: Globaler Parameter atvSymbolColor).
  • Tabellenfarben: Globale liste mit der Farbkonfiguration (Standard: Globlae Liste atvObjectDisplays.Tables.ColorConfig).
  • Benutzerdefiniertes Theme: Name des benutzerdefinierten Themes (Standard: atviseTheme)
  • Statushinweis: Spezifiziert die Darstellung des Status über eine definierte globale Liste (Standard: Globaler Parameter atvStatusIndication)
  • Symbol Zurückstellen: Symbol für zurückgestellte Alarme.
  • Symbol fü Nicht zurückgestellt: Symbol für nicht zurückgestellte Alarme.
  • Symbol Einmal Zurückstellen: Symbol für einmalig zurückstellte Alarme.
  • Symbol Befristet Zurückstellen: Symbol für befristet zurückstellte Alarme.
  • Symbol Unterdrückt: Symbol für unterdrückte Alarme.
  • Symbol fü Nicht unterdrückt: Symbol für nicht unterdrückte Alarme.
  • Symbol für Einschaltet: Symbol für eingeschaltete Alarme.
  • Symbol für Ausnschaltet: Symbol für ausgeschaltete Alarme.
  • Symbol Bestätigt: Symbol für bestätigte Alarme.
  • Symbol Nicht bestätigt: Symbol für nicht bestätigte Alarme.
< Optionen:
  • Datenmodus: Datenmodus für die Standardfilterung (Standard: continue)
  • Daten je Abfrage: Anzahl der Datensätze je Abfrage/Fortsetzungspunkt (Standard: 10000)
  • Datenlimitierung: Anzahl der maximal verarbeiteten Datensätze (Standard: 250000)
  • Detailmodus: Alarme, Werte oder beides anzeigen (Standard: all)
  • Paging: Aktivieren der Schaltflächen für das Blättern (Standard: true)
  • Filterleiste sichtbar: Einblenden der Filterleiste (Standard: true)
  • Filter mit reguläre Ausdrücke: Reguläre Ausdrücke für das Filtern verwenden (Standard: false)
  • Filter mit Platzhalter: Aktivierung des Platzhalters "*" in der Suchfunktion (Standard: false)
  • Filter Groß-/Kleinschreibung: Groß- und Kleinschreibung beim Filtern beachten (Standard: false)
  • Filter Mindesteingabe: Minimale Anzahl an Zeichen bevor gefiltert wird (Standard: 1)
  • Filter Verzögerung [ms]: Zeit bis die Filterung nach Eingabe gestartet wird (Standard: 1000)
  • Zeilenumbruch in Zelle: Zu langer Text wird innerhalb der Zeile umgebrochen, bei zu wenig Platz mit "..." als Endung abgeschnitten (Standard: false)
  • Spalten verschiebbar: Das Ändern der Spaltenreihenfolge wird ermöglicht (Standard: true)
  • Mehrfachselektion bei Klick: Die Mehrfachselektion ist bei Klick auf die gesamte Zeile möglich (Standard: true)
Menüoptionen:
  • Menü anzeigen: Das Menü wird angezeigt (Standard: true)
  • Titel benutzerdefiniertes Menü: Der Titel des benutzerdefinierten Menüs (Standard: "T{Export CSV}")
  • Spaltenauswahl im Menü: Das Ein- und Ausblenden von Spalten wird im Menü ermöglicht (Standard: true)
  • Pickermmenü Titel: Der Titel des Pickermenüs (Standard: "T{Columns}")
  • Menühöhe: Die Höhe des Menüs (Standard: 300)
  • Menübreite: Die Breite des Menüs (Standard: 250)
Standardfilter:

Hier ist es möglich, Standardwerte für die Filterfelder zu definieren, die die Filterung beim Starten der historischen Liste festlegen.

  • Starzeit: Wert in Sekunden von der aktuellen Zeit zurück (Standard: 60)
  • Min. Priorität
  • Quelle (Adresse)
  • Benutzer
  • Ereignistext
  • Wert
  • Archiv
  • Typ
  • Liveansicht (Standard: false)
  • Aggregat (Standard: Average)
  • Aggregat Intervall (Standard: 1)
  • Aggregat Einheit (Standard: m)
  • Kommentar
  • Gruppe
