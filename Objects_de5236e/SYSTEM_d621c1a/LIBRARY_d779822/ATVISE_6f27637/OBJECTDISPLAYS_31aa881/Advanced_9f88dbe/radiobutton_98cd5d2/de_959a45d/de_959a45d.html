Beschreibung

Der Radiobutton wechselt zwischen zwei Werten (wie die Schaltfläche "Schalter") und kann nur als Eingabe- oder Ausgabe-Steuerelement verwendet werden. Mehrere Radiobuttons können zu einer Radiobutton-Gruppe zusammengefasst werden, in der immer nur ein Radiobutton aktiviert werden kann.

Er kann in folgenden Modi verwendet werden:

  1. nur "base" festgelegt: In diesem Fall muss der "Wert" ebenfalls definiert werden.
Bei dieser Konfiguration wird "base" bei jedem Klicken auf "Wert" gesetzt. Der Radiobutton wird aktiviert, wenn der Wert von "base" mit dem von "Wert" identisch ist.
  1. "base" und "Zu setzende Adresse" festgelegt:
Bei dieser Konfiguration geschieht bei jedem Klicken Folgendes:
  • Der Radiobutton wird aktiviert, wenn der Wert von "base" mit dem von "Wert" identisch ist.
  • Beim Klicken wird "Zu setzende Adresse" auf "Wert" gesetzt.
  • Der Radiobutton selbst ändert seinen Zustand nur (aktiviert oder nicht), wenn "base" geändert wird (Feedback-Kanal).
  1. weder "base" noch "Zu setzende Adresse" festgelegt:
In diesem Fall wird der Trigger "valuechanged" mit dem Wert von "Wert" aktiviert. Über diesen Trigger kann ein Simple Dynamic zu diesem Radiobutton hinzugefügt werden, der auf den Trigger reagiert und einen neuen Trigger aktiviert. Dieser neue Trigger kann dann von anderen Elementen verwendet werden.

Mit Hilfe einer Statusanzeige in Form eines zusätzlichen Rahmens kann zudem auch ein negativer Status des aktuellen Wertes und ein zusätzlich definierter Alarm dargestellt werden. Die Farben und Blinkintervalle werden in globalen Listen definiert.

Parameter

Dieses Steuerelement hat die folgenden allgemeinen Parameter:

  • base: Je nachdem, ob "Zu setzende Adresse" definiert ist oder nicht, folgt das Verhalten dem vorstehenden Modus 1 oder 3.
  • Alarmadresse: Optional zusätzliche Adresse eines Alarms, welcher angezeigt werden soll.
  • Radiobutton Group ID: Wenn Sie mehrere Radiobuttons zu einer Gruppe zusammenfassen wollen, müssen Sie diesen Parameter für alle Radiobuttons der Gruppe auf denselben Wert setzen (z. B. myradiobuttongroup).
  • Beschriftung: Beschriftung des Radiobuttons
  • Wert: Wert, der definiert, wann der Radiobutton aktiviert ist
  • Statusüberwachung aktivieren: Schaltet die Anzeige des aktuellen Wertestatus ein oder aus.
  • Statustrigger feuern: Statt der Statusanzeige kann auch ein hier definierter Trigger global gefeuert werden.
Schrift:
  • Schriftart: Schriftart, die auf den Wert angewendet wird (Standard: Arial)
  • Größe: Schriftgröße, die auf den Wert angewendet wird (Standard: 12)
  • Farbe: Farbe, die auf den Wert angewendet wird (Standard: Globaler Parameter atvFontColor)
Darstellung:
  • diese Option vorwählen: Bei der Einstellung "true" wird der Radiobutton beim Laden aktiviert sein, andernfalls nicht. Dies gilt nur für den Fall, dass keine "base" und keine "Zu setzende Adresse" definiert sind.
  • Füllfarbe: Füllfarbe des Radiobuttons (Standard: Globaler Parameter atvFillColor)
  • Füllfarbe Selectbereich: Füllfarbe des Selectbereichs (Standard: weiß)
  • Füllfarbe wenn inaktiv: Füllfarbe, wenn nicht aktiv (Standard: dunkelgrau)
  • Symbol: Symbol, das bei aktiviertem Radiobutton angezeigt wird; weitere Symbole können ausgewählt werden aus "Object Displays (ATVISE)/Symbols/20x20 (Standard: disc)
  • Symbolfarbe: Farbe des zu verwendenden Symbols (Standard: Globaler Parameter atvSymbolColor)
  • Konturfarbe gedrückt: Farbe der Kontur, die bei Klicken auf den Radiobutton eingestellt wird (Standard: weiß)
  • Fokus Konturfarbe: Farbe der Kontur, die beim Fokus der Schaltfläche eingestellt wird (Standard: weiß)
  • Rahmenfarbe: Farbe des Rahmens um das Steuerelement (Standard: Globaler Parameter atvBorderColor)
  • Statushinweis: Spezifiziert die Darstellung des Status über eine definierte globale Liste (Standard: Globaler Parameter atvStatusIndication)
  • Alarmhinweis: Spezifiziert die Darstellung des Alarms überber eine definierte globale Liste (Standard: Globaler Parameter atvAlarmIndication)
Optionen:
  • Zu setzende Adresse: Je nachdem, ob "base" definiert ist oder nicht, folgt das Verhalten dem vorstehenden Modus 2 oder 3.
  • Konsistenzgruppe: Name der Konsistenzgruppe.
  • Tab-Index: Bestimmt die Reihenfolge, in der mit der Tabulatortaste zwischen den einzelnen Object Displays gewechselt werden kann
  • Tooltip: Zeigt eine Tooltip Box mit dem eingegebenen Text an (Standard HTML-Elemente können auch eingegeben werden)
Sicherheit:
  • Benötigtes Recht: Der Benutzer muss über dieses Recht verfügen, um diesen Radiobutton aktivieren zu können.
  • Aktivierungsadresse: Stimmt der Wert dieser Adresse mit dem "Aktivierungswert" überein, ist der Radiobutton aktiv, so dass er betätigt werden kann.
  • Aktivierungswert: Wert, von dem der Radiobutton aktiviert wird
HINWEIS: Die Aktivierung des Radiobuttons kann nur über das benötigte Recht konfiguriert werden oder nur über die Aktivierungsadresse und den Aktivierungswert oder über beide. Werden beide Einstellungen vorgenommen, muss der Benutzer das benötigte Recht besitzen und der Wert der "Aktivierungsadresse" muss mit dem "Aktivierungswert" identisch sein, andernfalls wird der Radiobutton nicht aktiviert.

Beispiel 1

Für den vorstehend beschriebenen Modus 2 gehen wir von den folgenden Parametern aus:
  • base: node_feedback
  • Wert: 1
  • Zu setzende Adresse: node_set
Weiterhin gehen wir davon aus, dass die Werte von "base" und "Zu setzende Adresse" auf 0 eingestellt sind.
  1. Bei diesen Einstellungen wird die Checkbox nicht aktiviert.
  2. Beim Klicken wird "node_set" auf 1 (= "Wert") gesetzt; der Radiobutton ist noch immer nicht aktiviert (der Wert von "base" weicht noch immer vom "Wert" ab).
  3. Sobald "base" auf 1 gesetzt ist (Feedback-Kanal von SPS), ist der Radiobutton aktiviert" (der Wert von "base" ist jetzt mit dem "Wert" identisch).

Beispiel 2

Für den vorstehend beschriebenen Modus 2 gehen wir für eine Gruppe von Radiobuttons von den folgenden Parametern aus:
  • Radiobutton 1:
    • base: node_feedback
    • Radiobutton Group ID: mygroup
    • Beschriftung: eins
    • Wert: 1
    • Zu setzende Adresse: node_set
  • Radiobutton 2:
    • base: node_feedback
    • Radiobutton Group ID: mygroup
    • Beschriftung: zwei
    • Wert: 2
    • Zu setzende Adresse: node_set
Weiterhin gehen wir davon aus, dass die Werte von "base" und "Zu setzende Adresse" auf 0 eingestellt sind.
  1. Bei diesen Einstellungen wird keine Checkbox aktiviert.
  2. Beim Klicken auf Radiobutton 1 wird "node_set" auf 1 (= "Wert") gesetzt; der Radiobutton 1 ist noch immer nicht aktiviert (der Wert von "base" weicht noch immer vom "Wert" ab).
  3. Sobald "base" von Radiobutton 1 auf 1 gesetzt ist (Feedback-Kanal von SPS), ist der Radiobutton 1 aktiviert" (der Wert von "base" ist jetzt mit dem "Wert" identisch).
  4. Beim Klicken auf Radiobutton 2 wird "node_set" auf 2 (= "Wert") gesetzt; der Radiobutton 1 ist noch immer aktiviert (der Wert von "base" weicht noch immer vom "Wert" ab).
  5. Sobald "base" von Radiobutton 2 auf 2 gesetzt ist (Feedback-Kanal von SPS), ist der Radiobutton 2 aktiviert" (der Wert von "base" ist jetzt mit dem "Wert" identisch).
