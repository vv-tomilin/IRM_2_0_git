Description

This display allows the user to load and save configurations to a node and also import and export configurations as json-files.

A node address has to be selected before a configuration can be saved or loaded. If there are any configurations already saved on the node they can be now loaded.

Options

Settings

  • Kompression: Determines whether the data is to be compressed when saving.

Configuration (save/load)

  • Index node: Defines the data point in which the configuration is saved.
  • Save Konfiguration: Saves the specified configuration.
  • Load or delete: Loading or deleting the selected configuration. (Reload, refresh the selection).
  • Create new: Creates a new configuration with the given names.

Export/Import

  • Export: Exports all configurations of the diagram to a json file.
  • Import: Imports all configurations for the diagram from a json file.
After one or more configurations are imported from a file, the configuration of your choice has to be loaded to be active.

Parameters

The following parameters define the rights for loading or saving a configuration:

  • necessary right: the user must have this right for the function to work
  • activation address: if the value of this address equals to "activation value", then the function will be active
  • activation value: value which activates the function
