Beschreibung

Dieses Steuerelement kann sowohl als Eingabe- & Ausgabeelement als auch als reines Ausgabeelement verwendet werden.

Darüber hinaus wird der Trigger "valuechanged" bei erfolgten Wertänderungen gefeuert.

Grundsätzlich können folgende Parametergruppen unabhängig vom Eingabemodus verwendet werden:

  • Schrift
  • Darstellung
  • Sicherheit
  • Optionen
Die verbliebenen Parametergruppen hängen vom jeweiligen Eingabemodus ab.

Eingabemodi

  1. Number: Der Eingabemodus "Number" erlaubt die Eingabe von "-", ",", ".", "0-9" und berücksichtigt zusätzlich folgende Parametergruppen:
    • Number Format
    • Limits
    • Enum List Modus
  2. Boolean: Der Eingabemodus "Boolean" erlaubt die Eingabe von "true", "false", "1", "0" und berücksichtigt zusätzlich folgende Parametergruppe:
    • Enum List Modus
  3. String: Der Eingabemodus "String" erlaubt die Eingabe aller Zeichen und berücksichtigt zusätzlich folgende Parametergruppe:
    • Enum List Modus
  4. Password: Der Eingabemodus "Password" erlaubt die Eingabe aller Zeichen berücksichtigt keine zusätzlichen Parametergruppen.

    5. Achtung: Falls ein Zeichen eingegeben wird, welches nicht dem Eingabemodus entspricht, dann wird eine Warnung in der Konsole ausgegeben und der Wert wird nicht gesetzt.

    Nähere Details zu den Parametergruppen befinden sich im nächsten Abschnitt.

Parameter

Achtung: Falls es sowohl die Möglichkeit gibt einen Wert direkt über einen Parameter zu setzen und auch über eine Adresse zu setzen, dann wird immer bei Angabe beider Werte, derjenige von der Adresse gesetzt.

  1. Ohne Parametergruppe
    • base: Diese Adresse wird gesetzt.
    • Alarmadresse: Optional zusätzliche Adresse eines Alarms, welcher angezeigt werden soll.
    • Eingabemodus: Hier werden die oben beschriebenen Eingabemodi bestimmt.
    • direkte Eingabe erlauben: Auswahl zwischen direkter Eingabe und Eingabe über SVG Tastatur.
    • Statusüberwachung aktivieren: Schaltet die Anzeige des aktuellen Wertestatus ein oder aus.
    • Statustrigger feuern: Statt der Statusanzeige kann auch ein hier definierter Trigger global gefeuert werden.

  2. Parametergruppe: Number Format
    • angezeigte Einheit: Diese Einheit wird zusätzlich zum Wert angezeigt.
    • Zeichen anzeigen: Bestimmt, ob das jeweilige Vorzeichen angezeigt werden soll oder nicht.
    • führende Nullen: Anzahl der Stellen links vom Dezimalkomma (Beispiel: Bei einer Einstellung von 3 wird der Wert 11 als 011 angezeigt).
    • Nachkommastellen: Anzahl der Stellen rechts vom Dezimalkomma.
    • Position des Kommas: Wandelt den Wert durch Dividieren durch 10 um (Beispiel: Bei einer Einstellung von 2 wird der Wert 2000 als 20,00 angezeigt).

  3. Parametergruppe: Enum List Modus
    • enum Liste: Globale Liste, die den anzuzeigenden Wert definiert.

    Ist eine "enum Liste" festgelegt, wird der Wert, sofern vorhanden, aus der globalen Liste entnommen.

    Beispiel:

    • Hinzufügen einer globalen Liste (z. B. "mylist") zu den globalen Parametern
    • Geben Sie "1" in die Spalte "Schlüssel" der ersten Zeile und einen zugehörigen Wert (z. B. 10) in die Spalte "Wert" der ersten Zeile ein.
    • eben Sie "2" in die Spalte "Schlüssel" der zweiten Zeile und einen zugehörigen Wert (z. B. 20) in die Spalte "Wert" der zweiten Zeile ein usw.
    • Wählen Sie Ihre oben definierte globale Liste (z. B. "mylist") als Wert des Parameters „enum Liste" aus.
    • Dadurch wird bewirkt, dass das "Ein-/Ausgabeelement Zahl" den Wert auf 10 setzt, wenn "base" = 1, bzw. auf 20, wenn "base"= 2, usw.

  4. Parametergruppe: Schrift
    • Schriftart: Schriftart, die auf den Wert angewendet wird (Standard: Arial).
    • Größe: Schriftgröße, die auf den Wert angewendet wird (Standard: 12).
    • Farbe: Farbe, die auf den Wert angewendet wird (Standard: schwarz).

  5. Parametergruppe: Darstellung
    • Füllfarbe: Füllfarbe (Standard: grau).
    • Füllfarbe nicht editierbar: Füllfarbe, wenn die Checkbox nicht editierbar ist (Standard: weiß).
    • Füllfarbe wenn inaktiv: Füllfarbe, wenn nicht aktiv (Standard: dunkelgrau).
    • Fokus Konturfarbe: Konturfarbe, wenn das Element den Fokus erhält.
    • Statushinweis: Spezifiziert die Darstellung des Status über eine definierte globale Liste (Standard: Globaler Parameter atvStatusIndication)
    • Alarmhinweis: Spezifiziert die Darstellung des Alarms überber eine definierte globale Liste (Standard: Globaler Parameter atvAlarmIndication)

  6. Parametergruppe: Limits
    • Unteres Limit unterschritten (off=kein Limit): Wert für Unteres Limit unterschritten (Standard: off, d. h. kein Limit)
    • Unteres Limit erreicht (off=kein Limit): Wert für Unteres Limit erreicht (Standard: off, d. h. kein Limit)
    • Oberes Limit erreicht (off=kein Limit): Wert für Oberes Limit erreicht (Standard: off, d. h. kein Limit)
    • Oberes Limit überschritten (off=kein Limit): Wert für Oberes Limit überschritten (Standard: off, d. h. kein Limit)
    • Farbe für Unteres Limit unterschritten: Füllfarbe , wenn " Unteres Limit unterschritten" >= Wert (Standard: rot).
    • Farbe für Unteres Limit erreicht: Füllfarbe , wenn Unteres Limit unterschritten" < Wert <= "Unteres Limit erreicht" (Standard: gelb).
    • Farbe für Oberes Limit erreicht: Füllfarbe , wenn " Oberes Limit erreicht" <= Wert < "Oberes Limit überschritten" (Standard: gelb).
    • Farbe für Oberes Limit überschritten: Füllfarbe , wenn " Oberes Limit überschritten" <= Wert (Standard: rot).
    • Minimum: Untere Grenze des Werts (Standard: off, d. h. kein Limit).
    • Maximum: Obere Grenze des Werts (Standard: off, d. h. kein Limit).
    • Adresse für Minimum: Adresse, der das Minimum entnommen wird; setzt den Parameter "Minimum" außer Kraft.
    • Adresse für Maximum: Adresse, der das Maximum entnommen wird; setzt den Parameter "Maximum" außer Kraft.
    • Adresse für Unteres Limit unterschritten: Adresse, der "Unteres Limit unterschritten" entnommen wird; setzt den Parameter "Unteres Limit unterschritten" außer Kraft.
    • Adresse für Unteres Limit erreicht: Adresse, der "Unteres Limit erreicht" entnommen wird; setzt den Parameter "Unteres Limit erreicht" außer Kraft.
    • Adresse für Oberes Limit erreicht: Adresse, der "Oberes Limit erreicht" entnommen wird; setzt den Parameter "Oberes Limit erreicht" außer Kraft.
    • Adresse für Oberes Limit überschritten: Adresse, der "Oberes Limit überschritten" entnommen wird; setzt den Parameter "Oberes Limit überschritten" außer Kraft.

  7. Parametergruppe: Sicherheit
    • editierbar: Bei der Einstellung "No" werden die Werte nur angezeigt, aber es kann nichts eingegeben werden (Standard: Yes).
    • Benötigtes Recht: Der Benutzer muss über dieses Recht verfügen, um dieses Steuerelement aktivieren, d. h. Werte eingeben zu können.
    • Aktivierungsadresse: Stimmt der Wert dieser Adresse mit dem "Aktivierungswert" überein, ist das Steuerelement aktiv, so dass es als Eingabefeld verwendet werden kann.
    • Aktivierungswert: Wert, von dem das Steuerelement aktiviert wird.

    Achtung: Wenn nur die Aktivierungsadresse bzw. der Aktivierungswert angegeben wird, dann ist das Steuerelement aktiv, da diese beiden Parameter nur gemeinsam funktionieren.


  8. Parametergruppe: Optionen
    • alternative Ausgabeadresse: Sofern festgelegt, wird nur diese Adresse gesetzt (nicht dagegen "base").
    • Konsistenzgruppe: Name der Konsistenzgruppe.
    • Tab-Index: Bestimmt die Reihenfolge, in der mit der Tabulatortaste zwischen den einzelnen Object Displays gewechselt werden kann.
    • Tooltip: Zeigt eine Tooltip Box mit dem eingegebenen Text an (Standard HTML-Elemente können auch eingegeben werden).
    • Text bei Fokus markieren: Markiert den Text, wenn die direkte Eingabe erlaubt ist.

    Achtung: Die alternative Ausgabeadresse funktioniert nur, wenn keine Konsistenzgruppe angegeben ist.
