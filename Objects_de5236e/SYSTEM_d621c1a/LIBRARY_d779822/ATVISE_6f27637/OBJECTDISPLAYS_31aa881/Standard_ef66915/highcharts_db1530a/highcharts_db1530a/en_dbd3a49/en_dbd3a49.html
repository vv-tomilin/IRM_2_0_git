Description

The chart display will be used to display the values as a chart.

It displays a chart with the raw values or aggregate values of a node. Configure the node by using the display parameters (see parameter Series [1..10]) or by using the configuration dialog of the toolbar. The display can be used alone or combined together with the toolbar and / or the legend display.

Zoom and pan:
Zooming is performed by selecting the desired area with the left mouse button or by pinching with two fingers on touch devices. To reset the zoom factor press the "Reset zoom" button on the right top. During zooming the chart can be panned over the time axis by pressing shift + left mouse button or by touch gestures.

Measurement marker:
Up to two measurement markers can be activated via the toolbar or the display parameters. To view the measured values the legend is required.

Notifications:
On the occurrence of errors and warnings (e.g. data source not available) an according symbol appears in the right top corner. On click on this symbol a notification window with details is shown.

Please note:
Aggregates are read from the archive(s) only once and the values will not be updated. If the option "non-stop" is activated, the last value is displayed in any case.

Parameters

The following common Parameters exists:

  • chart name: This is a unique name to identify the chart instance.
  • chart group: Multiple chart instances can be grouped.
  • chart mode: Defines the data mode, possible values: mixed (default), live and history.
  • skip default modules: De-/activates the loading of default highcharts modules.
  • additional modules: Defines additional modules to be loaded. (Example: "highcharts/modules/exporting.js,highcharts/datasource/opcua.js,highcharts/atviseModules/datahandler.js,highcharts/atviseModules/measuringcursor.js,highcharts/atviseModules/utils.js,highcharts/atviseModules/downsample.js")
  • configuration node: Node (address) which contains the predefined configuration.
  • configuration filename: File which updates a predefined configuration when saving is set in the file system.
  • configuration name: Name of the predefined configuration.
  • saving method: Determines whether configurations are stored in node(s) or the file system.
  • compression: Determines whether the configuration data is compressed when it is saved.
  • data source: Defines the datasource module which will be loaded for data handling (default: opcua).
  • chart title: Title of the chart.
  • animations: Turns animations off (default) or on..
  • measurement cursor 1: De-/activate measurement cursor 1.
  • measurement cursor 2: De-/activate measurement cursor 2.
  • tooltips: Enable or disable tooltips.
  • export menu: Enable or disable export menu button.
  • necassary right for export: The right needed for being able to export.
  • local time: If true the chart uses the local time, if false it uses UTC.
  • disable downsampling: Disable or enable downsampling.

Y-Axis *

The following parameters can be set for each y-axis:
  • id: ID of the axis.
  • title: Labeling of the axis.
  • visibility: Defines the visibility of the axis.
  • minimum: Lower threshold at which the data is displayed.
  • maximum: Upper threshold to which the data is displayed.
  • axis line width: Represented line weight of the axis (in pixels).
  • axis color: Represented color of the axis (hex code).
  • axis position: Position of the axis (left / right).
  • axis label: Display of the axis title (true / false).
  • axis label alignment: Horizontal alignment of the axis title (left, center or right).
  • label format: Define a format string for the axis label (see highcharts API reference).
  • crosshair enabled: Enabling the display of the crosshair (true, false).
  • crosshair snap: Activate the snap of the crosshair.
  • crosshair width: Line width of the crosshairs (in pixels).
  • crosshair color: Color of the crosshair (hex code).
  • crosshair type: Used line type of crosshair.
  • tick interval: Interval of the main grid for orientation.
  • minor tick interval: Interval of the auxiliary grid for orientation aid.
  • grid width: Line weight of the main grid lines (in pixels).
  • grid color: Color of the main grid lines (hex code).
  • minor grid width: Line width of the auxiliary grid lines (in pixels).
  • minor grid color: Color of the auxiliary grid lines (hex code).

X-Axis

The following parameters can be set for the x-axis:
  • title: Labeling of the axis.
  • timespan: Value of the time span to be displayed (without unit).
  • timespan unit: Unit of time to be displayed.
  • axis line width: Represented line weight of the axis (in pixels).
  • axis color: Represented color of the axis (hex code).
  • axis position: Position of the axis (left / right).
  • axis label: Display of the axis title (true / false).
  • axis label alignment: Horizontal alignment of the axis title (left, center or right).
  • crosshair enabled: Enabling the display of the crosshair (true, false).
  • crosshair snap: Activate the snap of the crosshair.
  • crosshair width: Line width of the crosshairs (in pixels).
  • crosshair color: Color of the crosshair (hex code).
  • crosshair type: Used line type of crosshair.
  • xAxis type: Type of axis scaling (default: datetime).
  • date time format: Format of the date output (default: "% H:% M:% S").
  • tick interval: Interval of the main grid for orientation.
  • minor tick interval: Interval of the auxiliary grid for orientation aid.
  • grid line width: Line weight of the main grid lines (in pixels).
  • grid line color: Color of the main grid lines (hex code).
  • minor grid width: Line width of the auxiliary grid lines (in pixels).
  • minor grid color: Color of the auxiliary grid lines (hex code).

Series * (data set)

The following parameters can be set for each series:
  • address: Node address of the value to be displayed.
  • interpolation: Interpolation of values (line, spline, step).
  • name: Name of the series.
  • data archive: Name of archive that will be used for displaying historical values (default: "").

    • This parameter is optional and particularly relevant for Portal users and custom WebMI implementations, using more more then one data archive.
  • line width: Line width for displaying the values (in pixels).
  • color: Color for the representation of the values (hex code).
  • dash style: Linetype for the representation of the values.
  • marker: Activate data point markers.
  • visibility: Visibility of the presentation at the initialization of the chart.
  • nonstop: Continuous presentation of missing values.
  • aggregate: Filter for aggregates of the specified address.
  • aggregate interval: Interval of the aggregate.
  • aggregate unit: Unit of aggregate.
  • x-axis: ID of the assigned x-axis.
  • y-axis : ID of the assigned y-axis

Example

Create an empty display and add this display by drag & drop. Set the desired addresses that should be displayed in the chart.

This display can be used stand alone with given parameters or together with toolbar and / or legend display, therefore the parameter "chartname" must be unique and equal on all trend displays that are in use.
